# 1. 專案概覽

**英文小勇士（English Warrior）** 是一款面向台灣中小學生設計的英文文法練習 Web 遊戲。以宇宙星空為主題，透過四個循序漸進的關卡，讓學生在遊戲化情境中反覆練習英文文法，並支援三種難度以因應不同程度的學習者。

| 項目       | 說明                                                   |
|:-----------|:-------------------------------------------------------|
| 專案類型   | 純前端網頁遊戲（Static Web App）                       |
| 部署方式   | GitHub Pages                                           |
| 技術棧     | HTML5 / CSS3 / Vanilla JavaScript（無框架）            |
| 目標受眾   | 台灣國中小學生，以及需要準備全民英檢初級的學習者       |
| 語言支援   | 繁體中文介面 + 英文題目                                |
| 檔案數量   | 5 個 HTML 檔案（index + 4 關卡）                       |
| 部署網址   | <https://lorryleou.github.io/English-game/>            |

# 2. 頁面結構與檔案說明

專案共包含以下 HTML 檔案，每個檔案皆為獨立的單頁應用（SPA），包含完整的 HTML、CSS 及 JavaScript，無需額外引入外部 JS 函式庫。

| 檔案名稱       | 功能說明 |
|:---------------|:---------|
| `index.html`   | 主選單頁面。顯示四個關卡入口卡片，以及難度選擇切換器（三選一：基礎版／高階版／英檢初級挑戰）。點擊關卡卡片時，將所選難度以 URL 參數（`?difficulty=basic/advanced/gept`）傳遞至各關卡頁面。 |
| `level1.html`  | 關卡 1：克漏字填空。每題提供句子前段 + 空格 + 句子後段，並給出 4 個選項，玩家選出正確的動詞形式、介系詞等。每局從題庫中隨機抽取 10 題。 |
| `level2.html`  | 關卡 2：句子重組（Rearrange）。玩家點選散落的單字晶片，依序組成正確的句子。每局隨機抽取 10 題。（此關卡於本次規劃中暫不擴充題庫） |
| `level3.html`  | 關卡 3：對話情境選擇。提供對話情境（可選帶有場景描述的 context box），玩家從 3 個選項中選出最正確的英文回應。每局使用全部題庫題目。 |
| `level4.html`  | 關卡 4：終極闖關（Boss Challenge）。混合三種題型（填空、重組、對話），加上倒數計時器與生命值系統（❤️ × 3），每局隨機抽取 12 題。 |

# 3. 關卡設計詳細說明

| 關卡   | 遊戲類型                        | 題目數（每局）     | 難度系統         | 計分方式               |
|:-------|:-------------------------------|:-------------------|:-----------------|:-----------------------|
| 關卡 1 | 克漏字填空（Fill-in-the-blank） | 每局 10 題（隨機） | 三難度各獨立題庫 | +10 分／題，答對即得分 |
| 關卡 2 | 句子重組（Rearrange）           | 每局 10 題（隨機） | 三難度各獨立題庫 | +10 分／題，答對即得分 |
| 關卡 3 | 對話情境選擇（Dialogue）        | 全部題庫（不抽題） | 三難度各獨立題庫 | +10 分／題，答對即得分 |
| 關卡 4 | 混合三題型 + 限時 + 生命值      | 每局 12 題（隨機） | 三難度混合題庫   | +10 分／題，答錯扣命   |

## 3.1 關卡 1：克漏字填空

每題資料結構包含：

- `before`：空格前方的句子片段
- `blank`：正確答案
- `after`：空格後方的句子片段
- `options`：4 個選項（陣列）
- `hint`：中文提示（以 💡 開頭）
- `explanation`：答題後顯示的文法說明

選項會在渲染時自動打亂順序（Fisher-Yates shuffle）。答題後顯示正確答案與文法說明，並以顏色標記對錯（綠色／紅色）。

## 3.2 關卡 3：對話情境選擇

每題資料結構包含：

- `context`（可選）：場景描述文字，例如「📞 情境：你打電話到診所…」
- `dialogue`：對話陣列，每筆包含 `speaker`（A/B）、`avatar`（emoji）、`name`、`text`、`isQuestion`（是否為填答位置）
- `options`：3 個選項，每個含 `text` 與 `correct` 屬性
- `explanation`：答題後顯示的文法說明

GEPT 英檢初級挑戰等級的對話題，一律附有 `context` 場景設定，模擬真實生活情境（如：電話預約、超市問路、飯店確認訂房等）。

## 3.3 關卡 4：混合闖關

同時具備三種題型（Fill／Rearrange／Dialogue），題庫在 Level 4 中分為九個子陣列：

- `FILL_BASIC` / `FILL_ADVANCED` / `FILL_GEPT`
- `REARRANGE_BASIC` / `REARRANGE_ADVANCED` / `REARRANGE_GEPT`
- `DIALOGUE_BASIC` / `DIALOGUE_ADVANCED` / `DIALOGUE_GEPT`

根據 `difficulty` URL 參數，選取對應的三個子陣列合併後，隨機抽取 12 題。計時器秒數根據難度調整：

| 難度         | 計時秒數 |
|:-------------|:---------|
| 基礎版       | 15 秒    |
| 高階版       | 20 秒    |
| 英檢初級挑戰 | 25 秒    |

# 4. 三難度系統

每個關卡均支援三種難度，由 `index.html` 透過 URL 參數（`?difficulty=`）傳遞至各關卡頁面。各關卡在頁面載入時讀取此參數，切換對應題庫並套用對應的視覺主題。

| 難度名稱              | URL 參數值 | 難度程度 | 語法範疇                                             |
|:---------------------|:-----------|:---------|:-----------------------------------------------------|
| 🌱 基礎版            | `basic`    | 一般     | be 動詞、助動詞、現在／過去簡單式                     |
| ⚡ 高階版            | `advanced` | 較難     | 現在完成式、被動語態、附加問句、情態完成式             |
| 🎯 英檢初級挑戰      | `gept`     | 挑戰     | 完成進行式、過去完成式、虛擬語氣、複雜句型             |

# 6. 遊戲機制說明

## 6.1 計分系統

- 每答對一題：**+10 分**
- 每答錯一題：**0 分**（關卡 1～3），**扣一條生命值**（關卡 4）
- 時間到未作答（關卡 4）：視為答錯，扣生命值
- 最終分數 = 答對題數 × 10

## 6.2 關卡 4 生命值系統

- 初始生命值：❤️❤️❤️（三條命）
- 答錯或超時依序扣除：❤️❤️🖤 → ❤️🖤🖤 → 💀
- 三條命全扣完：遊戲立即結束，顯示 Game Over 畫面
- 計時器秒數：基礎版 15 秒、高階版 20 秒、英檢初級挑戰 25 秒

## 6.3 視覺回饋

- **答對：** 綠色邊框 + 正確標示 + 紙花雨（confetti）動畫
- **答錯：** 紅色邊框 + 答錯標示 + 背景紅色閃爍
- **答題後：** 顯示文法說明（`explanation`），輔助學習
- **結果畫面：** 顯示分數、答對／答錯題數、正確率，並根據正確率給出不同的評語

## 6.4 隨機出題邏輯

使用 Fisher-Yates shuffle 演算法打亂題庫，每局從打亂後的題庫中取前 *N* 題（*N* 依各關卡設定）。選項也會在渲染時重新打亂，避免答案位置固定。

# 7. 技術實作細節

## 7.1 難度傳遞機制

`index.html` 使用 JavaScript 在導向時附加 URL 參數：

```javascript
window.location.href = `level1.html?difficulty=${selectedDifficulty}`;
```

各關卡頁面讀取方式：

```javascript
const params = new URLSearchParams(window.location.search);
const difficulty = params.get('difficulty') || 'basic';
```

讀取後，根據 `difficulty` 值切換題庫，並在 `body` 元素上加上對應的 CSS class（`advanced` / `gept`），觸發不同的視覺主題。

## 7.2 CSS 主題切換

所有視覺差異都透過 CSS class 選取器實現。基礎版為預設樣式，高階版與英檢初級挑戰分別透過 `body.advanced` 與 `body.gept` 覆蓋樣式。所有 CSS 內嵌於各 HTML 檔案的 `<style>` 標籤中，無需外部 CSS 檔案。

## 7.3 關鍵注意事項

::: {.callout-warning}
以下為開發時容易踩到的坑，請特別留意。
:::

1. **JavaScript 字串中的撇號（apostrophe）問題**
   若題目 `name` 欄位含有 `'s`（如 `Jason's Mom`），外層引號必須使用雙引號（`"Jason's Mom"`），否則會造成 JS 語法錯誤，導致整個頁面的腳本無法執行。

2. **`context-box` 的顯示控制**
   Level 3 的 GEPT 題庫使用 `context` 欄位提供場景說明，渲染時需偵測 `q.context` 是否存在，有則顯示，無則隱藏（`display:none`）。

3. **選項打亂後的正確答案對應**
   Level 3 的對話題選項打亂後，`checkAnswer` 函式必須用打亂後的陣列（shuffled）作為索引依據，不能用原始陣列順序。

4. **Level 4 題型判斷**
   題目物件必須有 `type` 欄位（`'fill'` / `'rearrange'` / `'dialogue'`），`renderQuestion` 函式依此分派到不同的渲染函式。
